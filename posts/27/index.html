Git Status of My Octopress Folder

Mar 1st, 2014 5:32 pm | Comments | ∞

I‚Äôd like to include my GitHub repositories here. To do so, it‚Äôs a piece of cake to slightly modify _config.yml.1

Once, I curiously typed git status in ~/octopress. The following screenshot is from my terminal.

Then I blindly followed the ‚Äúadvice‚Äù of git status and typed

$ git add .

It seemed that I had made a mistake. Actually, without adding those modified and untracked files on the stage for commit, the following two commands will do update the contents on the GitHub pages.

$ rake generate
$ rake deploy

The final step is to view your blog again after going out for a cup water or some snacks.

How about the git status? Simply forget about it!

(Last added on MAR 15TH, 2014)

If you have more than one local repository of your Octopress or Jekyll site, you may upload your contents to the Internet as long as the corresponding branch in the local repository is newer than the one in the remote repository. To keep the Octopress or Jekyll code updated, one needs to use Git to check if the source branch contains the latest changes. Otherwise, after writing a new post, it is possible that one can‚Äôt upload the new post to the Octopress or Jekyll site using rake deploy.

Thus, eight days ago, I made Git commit 8c7be99.

I tried typesetting a $\rm \LaTeX$ Beamer slide using pandoc markdown since it‚Äôs easy to use. According to the official demo1, with a CSL file2, which specifies the style to the citation, users are suppose to be able to create a document with a bibliography. However, when I ran the following command in Vim on Ubuntu¬†12.04 LTS, pandoc hanged.

:!pandoc -s -S --biblio mybib.bib --csl foo.csl -t beamer %\
-V theme:Berlin -o %<.pdf

I admit that it‚Äôs too complicated. I tested several things:

  1. $\rm \LaTeX$ Beamer class
  2. Bibliography

To figure out which part of the command is wrong, I tried to remove the part related to item 1. Issuing a simple command for item 2, the usage of one of the CPU in my computer rose to almost 100%, and remained high with small fluctuations.

This issue is indeed one faced by the community of Ubuntu users, so I can find official advices on GitHub. Eventually, the last comment on the page has solved my problem.
